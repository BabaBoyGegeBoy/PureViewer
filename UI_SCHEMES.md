# PureViewer UI 重构方案文档

> 保留此文档以便随时回退到其他方案。当前采用 **方案 A**。

---

## 方案 A：悬浮自动隐藏（当前实现）

### 设计理念
图片占满整个窗口，工具栏半透明悬浮叠加在图片上方，鼠标不动 2 秒后自动隐藏，移动鼠标重新浮现。类似视频播放器的控制栏体验。

### 布局结构
```
┌──────────────────────────────────────────────┐
│ ▫ 悬浮顶栏（半透明 #CC222222）                │  ← 选择文件夹 | 含子文件夹 | 搜索 | 标签 | 路径信息
│   2秒无鼠标操作自动隐藏                       │     鼠标移入顶栏区域时保持显示
│                                              │
│                                              │
│                                              │
│            图片占满整个窗口                    │  ← Stretch=Uniform, 背景 #FF1E1E1E
│            背景 #FF1E1E1E                     │
│                                              │
│                                              │
│                                              │
│ ▫ 悬浮底栏（半透明 #CC222222）                │  ← ◀ ▶ 全屏 退出全屏 缩略图 100%
│   2秒无鼠标操作自动隐藏                       │     鼠标移入底栏区域时保持显示
└──────────────────────────────────────────────┘
 左侧：作者侧栏按 Tab 键滑入/滑出（半透明 #CC222222）
```

### 技术实现要点
- **布局**: 根容器改为 `Grid`（非 DockPanel），所有 UI 层叠加在图片上
- **层叠顺序**（Z-Order，先声明在下层）:
  1. 图片层（ScrollViewer + Image）— 占满整个 Grid
  2. 缩略图网格层 — 占满整个 Grid
  3. 侧栏层 — DockPanel.Dock=Left, 半透明背景
  4. 顶栏层 — VerticalAlignment=Top, 半透明背景
  5. 底栏层 — VerticalAlignment=Bottom, 半透明背景
  6. 加载/缩放提示层 — 居中/右下
- **自动隐藏**:
  - `DispatcherTimer` 2秒间隔
  - `Window.MouseMove` 重置计时器，设置顶栏/底栏 `Visibility=Visible`
  - 计时器 Tick 时设置顶栏/底栏 `Visibility=Collapsed`
  - 顶栏/底栏自身 `MouseEnter` 暂停隐藏，`MouseLeave` 恢复计时
- **侧栏**: `Tab` 键切换 `Visibility`，滑入/滑出用 `TranslateTransform` + `DoubleAnimation`
- **快捷键**: Tab 切换侧栏，其他不变
- **背景色**: 图片区域 `#FF1E1E1E`（更深黑），工具栏 `#CC222222`（80%透明黑）

### 优势
- 图片 100% 占满窗口，最沉浸式阅读体验
- 工具栏不遮挡操作，需要时自动出现
- 与全屏模式体验一致（无需全屏也沉浸）

### 代价
- 自动隐藏逻辑增加代码复杂度
- 需处理鼠标事件优先级（工具栏 vs 图片拖拽）
- 搜索框/标签下拉与自动隐藏的交互需仔细处理

## 方案 B：极简栏+折叠

### 设计理念
保留一行极简顶栏（仅路径信息+关键操作），底栏默认折叠仅 hover 展开，侧栏按键切换。图片区域明显增大但不完全占满。

### 布局结构
```
┌──────────────────────────────────────────────┐
│ [📁] 共 56/230 张 - C:\path | 🔍 [___] | 🏷  │  ← 极简顶栏，28px 高
├──────────────────────────────────────────────┤
│                                              │
│                                              │
│            图片区域                           │  ← 比方案A少一行+底栏空间
│            (最大化但非全屏)                    │
│                                              │
│                                              │
│ ┌─ hover 展开的底栏 ──────────────────────┐  │
│ │  ◀ ▶ 全屏 退出全屏 缩略图 100%          │  │
│ └──────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
 左侧：作者侧栏按 Tab 键切换（不透明，固定宽度）
```

### 技术实现要点
- 顶栏固定 28px 高，用图标按钮替代文字按钮节省空间
- 底栏用 `Expander` 或 `Popup`，默认 Collapsed，hover/点击展开
- 侧栏：Tab 键切换 Visibility
- 图片区域高度 = 窗口高度 - 28px

### 优势
- 改动量小，基于现有布局微调
- 始终可见路径信息，不需要移动鼠标就知道状态
- 交互简单直接，不需要计时器

### 代价
- 顶栏仍占用一行空间
- 底栏展开时压缩图片区域
- 视觉沉浸感不如方案 A

## 方案 C：纯沉浸无固定 UI

### 设计理念
完全无固定 UI 元素，所有控件通过右键菜单或鼠标移动时弹出的浮动面板访问。图片完全占满窗口。

### 布局结构
```
┌──────────────────────────────────────────────┐
│                                              │
│                                              │
│                                              │
│            图片占满整个窗口                    │  ← 无任何可见 UI
│            背景 #FF1E1E1E                     │
│                                              │
│                                              │
│                                              │
│                                              │
│                                    100% ─────│  ← 仅右下角缩放指示
└──────────────────────────────────────────────┘

右键菜单：
  ├── 选择文件夹
  ├── 含子文件夹 ✓
  ├── ─────────
  ├── 上一张 / 下一张
  ├── 全屏 / 退出全屏
  ├── 缩略图视图
  ├── ─────────
  ├── 搜索... (弹出搜索框)
  ├── 标签筛选 (弹出下拉)
  └── 作者列表 (弹出侧栏)

鼠标移动到边缘：弹出对应浮动面板
  - 顶部边缘 → 搜索+标签面板
  - 左侧边缘 → 作者列表
  - 底部边缘 → 导航控制栏
```

### 技术实现要点
- 无任何固定 UI，所有控件在 `Popup` / `ContextMenu` 中
- `Window.MouseMove` 检测鼠标是否在边缘区域（50px 内），触发对应浮动面板
- 右键菜单使用 WPF `ContextMenu`
- 搜索框用 `Popup` 弹出
- 作者列表用 `Popup` + `ListBox`

### 优势
- 图片 100% 占满，零 UI 侵入
- 最极简的视觉体验
- 所有操作通过右键/快捷键，无视觉干扰

### 代价
- 交互发现性差，新用户不知道怎么操作
- 需要记住快捷键/右键菜单
- 浮动面板的定位和交互复杂
- 搜索等高频操作需要多次点击
